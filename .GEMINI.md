## 專案憲章 (Project Constitution)

# Climber — 專案憲章

## 願景 (Vision)
使用 AI Studio (Gemini 3.0) 建構的攀岩伴侶應用程式，協助攀岩者追蹤路線、進度與目標。此專案已轉型為 Web 應用程式 (React)，並整合 GCP 後端。

## 核心原則 (Core Principles)
- **AI 優先 (AI-First)**：利用 Gemini 3.0 進行智慧路線建議與進度分析。
- **現代化 Web 體驗**：使用 React 與原生 CSS 建立動態且美觀的介面。
- **資料完整性 (Data Integrity)**：準確追蹤攀岩記錄、難度與長期表現。
- **GCP 後端整合**：使用 Google Cloud Platform (GCP) 進行資料持久化與擴展。

## 開發指引 (Development Guidelines)
- 元件應保持小巧且專注。
- 使用 Vite + React 建構前端。
- 優先使用正體中文 (zh-TW) 進行文件記錄。
- 透過 Agent DevelopKit 或 Semantic Kernel JS 整合 AI 邏輯。
- 所有功能必須具備可測試性並文件化。

## 品質標準 (Quality Standards)
- 程式碼必須易讀且具備良好的註釋。
- UI 必須符合 WCAG 2.1 AA 無障礙標準。
- 性能：初始載入時間應低於 2 秒。
- 確保在 GCP 環境下運作穩定。

---

## 技術棧 (Tech Stack)
- 前端：React 18.x (Vite)
- AI：Gemini 3.0 (Flash)
- 後端：GCP (Cloud Run / Firestore)
